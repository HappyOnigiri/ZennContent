# はじめに：心機一転、からの「つらぽよ」

エンジニアとして業務はそれなりにがんばってきたつもりでした。
でもふと振り返ると、個人としてアピールできる技術資産が何もないことに気づいたんです。OSS も技術記事もゼロ。アピールできる「個人の実績」が空欄。

「今年こそは個人の技術資産を積み上げていこう」

そう決意して、技術発信用の新しいアカウントを作ることにしました。
これまで持っていたのはゲーム用アカウントと本名アカウントだけ。技術発信には向かないので、ハンドルネームを決めてイチから各種プラットフォームに登録していくことに。

……が、そんな私を待っていたのは、現代のインターネットが抱える「ゼロトラスト」という名の分厚い壁でした。

結果から言うと、**GitHub、Discord、Dev.to の3つのプラットフォームで、約2ヶ月の間に連続で凍結される**という、呪われているとしか思えないコンボを食らいました。

エンジニア歴の長い私でも「つらぽよ…」と天を仰ぐレベルの理不尽デスゲーム。
駆け出しエンジニアなら1ミリ秒で心が折れてIT業界から去りかねないこの「2026年の新規アカウント作成サバイバル」を、自戒と注意喚起を込めて共有します。

---

## 第1の試練：GitHub（テストデータの一発レッド）── 1月10日

年明け早々、コードを管理するために GitHub の新規アカウントを作成しました。
「さぁ、ここから始めるぞ！」という清々しい気持ちで最初のリポジトリを作成しプッシュを実行。

すると、突如としてアカウントが凍結状態に。

えっ？

どうやら、プッシュしたテストデータの中に GitHub の自動フィルターに引っかかる「NGワード」が含まれていたようです。
何のテストデータかって？ ……察してください。お願いします。健全な技術記事なので詳細は伏せますが、**大人の事情が詰まったデータ**だったとだけ申し上げておきます。

新年最初のプッシュが一発レッドカード。GitHub 側からすれば「新規アカウントがいきなり不審なデータを上げてきた」わけで、そりゃ撃たれますよね。

すぐにサポートへ英語でメールを送りました。要約すると以下のような内容です。

- 該当リポジトリにサンプルデータを含めてしまったのは完全なミスであること
- 自分は正当な開発者であり、規約違反の意図はないこと
- アカウントを復旧してもらえれば、該当ファイルまたはリポジトリごと即座に削除すること

ポイントは **「意図的ではない」「すぐに是正する」** という姿勢を明確に伝えることです。
約1週間で無事に解除されました。テストデータの中身には気をつけようね……。

## 第2の試練：Discord（即時凍結からの物理殴り）── 1月29日

GitHub の傷が癒えたところで、次は情報収集やコミュニティ参加のために Discord アカウントを作成しました。

アカウント登録完了！ → **即、凍結**。

嘘だろおい。

スパム bot 対策が極限まで高まっている昨今、新規 IP や作成直後の素早い操作は即 bot 判定されるようです。
また、当時は VPN を接続した状態でアカウントを作成していました。確証はありませんが、VPN 経由の IP アドレスはスパム業者が多用するため、これも即凍結の一因になった可能性があります。

サポートに解除申請を出し、待つこと1週間。ようやく解除された！
……と思いきや、解除後1分で「電話番号認証」を求められ、認証するまで一切の操作が不可能に。

せっかく1週間待ったのに、解除された瞬間にまた壁が立ちはだかるこの絶望感。
もうね、笑うしかない。

手持ちの電話番号はゲーム用の別アカウントに紐づいていたため使えません。「じゃあゲーム用アカウントから電話番号を外して付け替えれば……」とも考えましたが、ゲーム用アカウントにログインできなくなるリスクや、電話番号の付け替え自体が怪しい行動と判定されてさらに面倒になる未来が見えたので、やめました。

結局、**[povo](https://povo.jp/) で新しい電話番号を契約して物理で殴り勝つ** という脳筋プレイで解決。povo なら年間数百円で電話番号を維持できるので、こういう「認証用にもう1回線ほしい」という場面では最適解です。

SIM 契約という課金で BAN を突破するエンジニア、ロックすぎる。

## 第3の試練：Dev.to（スパム業者と間違えられた悲劇）── 2月28日

2連続 BAN を乗り越え、「さすがにもう大丈夫だろ」と油断していた私。
英語圏向けの技術ブログとして Dev.to にアカウントを作り、最近検証していた AI ツール（[Tenbin AI](https://tenbin.ai/)）の紹介記事を書きました。

記事を公開してダッシュボードを開くと、画面いっぱいの「404 PAGE NOT FOUND」。
警告メールすら一切なく、**「宣伝・スパム業者」として即時凍結**。

そんな……。

3回目になるとさすがに慣れてきた自分が怖い。

原因はおそらく、宣伝と受け取られかねない表現をしてしまったこと。記事タイトルが「I Stopped Paying $40/Month for AI...（AI に月40ドル払うのをやめた理由）」という、**英語圏のアフィリエイトスパム業者のド定番テンプレート** そのものだったのです。
Tenbin AI からお金なんて1円ももらっていない、純粋な技術レビューだったんですが……。
Dev.to はスパムに対してゼロトレランス（容赦なし）なため、人間が確認する前にシステムが即時 BAN を下します。

GitHub のときと同じ戦法でいこうと、すぐにサポートへメールで異議申し立てを送りました。

- スパムや宣伝の意図は一切なく、純粋に技術的な体験を共有したかったこと
- 該当記事がガイドラインに抵触していたなら、削除・修正に応じること
- 今後はガイドラインを厳守すること

**誠意をもって「意図的ではない」「是正する」** と伝えるアプローチ。GitHub では1週間で復旧できたので、今回もいけるだろうと思っていました。

……が、1ヶ月経っても運営からの返信は一切なく、「もう無理かも」と半ば諦めていました。

ところが、**凍結から約2ヶ月後にようやく返信が届き、無事に解除されました。**

運営からのメールによると、自動スパムフィルターの誤検知だったとのこと。「あなたのコンテンツは利用規約にもコミュニティガイドラインにも違反していませんでした」と明言され、アルゴリズムの改善にも取り組むと書かれていました。

Dev.to のサポート、ちゃんと人間がいた……！ 時間はかかったけど、誠実な対応に感謝です。

## まとめ：現代のプラットフォームは「信用スコアゼロ（マイナス）」から始まる

今回の一連の事件から得られた教訓は以下の通りです。

1. **新規アカウントは常に「bot・スパム」と疑われている**
   今の Web サービスはどこもスパム業者との終わらない戦争状態にあります。作成直後のアカウントは最も警戒されており、少しでも怪しい挙動（NGワード、怪しい通信経路、煽りタイトル）があれば即座にシステムに撃たれます。
2. **システムに撃たれても人格は否定されていない**
   「警告なしで凍結」されると心が折れそうになりますが、これは自動システムの誤検知です。あなたが悪いんじゃない。bot が悪い。
3. **冷静に「人間のサポート」を頼る**
   異議申し立てをして、生身のエンジニアであることを証明できれば、アカウントは高確率で復旧します。~~ただし Dev.to、テメーはダメだ( ꐦ◜ω◝ )~~ Dev.to さん、2ヶ月かかっても誠実に対応してくれました。さすがっす！ 諦めずに待つことも大事。

もし今、理不尽なアカウント凍結に悩んでいる駆け出しエンジニアの方がいたら、伝えたい。
**「あなたのせいじゃない。そして、歴の長いエンジニアでも普通に食らっているから安心してほしい」** と。

あと、テストデータの中身には本当に気をつけようね。本当に。
