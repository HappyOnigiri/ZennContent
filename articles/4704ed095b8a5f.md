## はじめに

エンジニアとして業務はそれなりにがんばってきたつもりでした。
でもふと振り返ると、個人としてアピールできる技術資産が何もない。OSS も技術記事もゼロ。

「今年こそは個人の技術資産を積み上げていこう」

そう決意して、技術発信用の新しいアカウントを作ることにしました。
これまで持っていたのはゲーム用アカウントと本名アカウントだけで、技術発信には向かない。ハンドルネームを決めてイチから各種プラットフォームに登録していくことにしたんですが、ここから約2ヶ月で GitHub、Discord、Dev.to の3つに連続で凍結されました。

呪われているとしか思えないコンボです。
その一部始終を書いておきます。

---

## GitHub：テストデータの一発レッド（1月10日）

年明け早々、コードを管理するために GitHub の新規アカウントを作成しました。
最初のリポジトリを作ってプッシュを実行。

いきなりアカウントが凍結状態に。

えっ？

プッシュしたテストデータの中に、GitHub の自動フィルターに引っかかる NG ワードが含まれていたようです。
何のテストデータかって？ ……察してください。健全な技術記事なので詳細は伏せますが、大人の事情が詰まったデータだったとだけ申し上げておきます。

新年最初のプッシュが一発レッドカード。
GitHub からすれば、新規アカウントがいきなり不審なデータを上げてきたわけで、そりゃ撃たれますよね。

すぐにサポートへ英語でメールを送りました。

- 該当リポジトリにサンプルデータを含めてしまったのは完全なミスであること
- 自分は正当な開発者であり、規約違反の意図はないこと
- アカウントを復旧してもらえれば、該当ファイルまたはリポジトリごと即座に削除すること

意図的ではないこと、すぐに是正すること。この2点をはっきり伝えるのが大事です。
約1週間で無事に解除されました。テストデータの中身には気をつけようね……。

## Discord：即時凍結からの物理殴り（1月29日）

GitHub の傷が癒えたところで、次は情報収集やコミュニティ参加のために Discord アカウントを作成しました。

アカウント登録完了。即、凍結。

嘘だろおい。

スパム bot 対策が極限まで高まっている昨今、新規 IP や作成直後の素早い操作は即 bot 判定されるようです。
当時は VPN を接続した状態でアカウントを作成していました。確証はありませんが、VPN 経由の IP アドレスはスパム業者が多用するため、これも即凍結の一因だった気がします。

サポートに解除申請を出し、待つこと1週間。ようやく解除された！
……と思いきや、解除後1分で電話番号認証を求められ、認証するまで一切の操作が不可能に。

せっかく1週間待ったのに、解除された瞬間にまた壁。もうね、笑うしかない。

手持ちの電話番号はゲーム用の別アカウントに紐づいていたため使えません。ゲーム用アカウントから電話番号を外して付け替えることも考えましたが、ログインできなくなるリスクや、電話番号の付け替え自体が怪しい行動と判定される未来が見えたので、やめました。

結局、[povo](https://povo.jp/) で新しい電話番号を契約して物理で殴り勝つという脳筋プレイで解決。povo なら年間数百円で電話番号を維持できるので、認証用にもう1回線ほしい場面では最適解です。

SIM 契約という課金で BAN を突破するエンジニア、ロックすぎる。

## Dev.to：スパム業者と間違えられた（2月28日）

2連続 BAN を乗り越え、さすがにもう大丈夫だろと油断していました。
英語圏向けの技術ブログとして Dev.to にアカウントを作り、最近検証していた AI ツール（[Tenbin AI](https://tenbin.ai/)）の紹介記事を書きました。

記事を公開してダッシュボードを開くと、画面いっぱいの「404 PAGE NOT FOUND」。
警告メールすら一切なし。宣伝スパム業者として即時凍結されていました。

そんな……。3回目になるとさすがに慣れてきた自分が怖い。

原因はおそらく記事タイトルです。
「I Stopped Paying $40/Month for AI...」という、英語圏のアフィリエイトスパム業者がよく使うド定番テンプレートそのものでした。Tenbin AI からお金なんて1円ももらっていない、純粋な技術レビューだったんですが……。
Dev.to はスパムに対してゼロトレランス（容赦なし）なので、人間が確認する前にシステムが即 BAN を下します。

すぐにサポートへメールで異議申し立てを送りました。スパムの意図はないこと、記事がガイドラインに抵触していたなら修正に応じること、今後はガイドラインを厳守すること。GitHub のときと同じで、誠意をもって是正の姿勢を伝える方針です。

GitHub では1週間で復旧できたので、今回もいけるだろうと思っていました。

……が、1ヶ月経っても運営からの返信は一切なく、もう無理かもと半ば諦めかけていました。

ところが、凍結から約2ヶ月後にようやく返信が届き、無事に解除。
運営からのメールによると、自動スパムフィルターの誤検知だったそうです。コンテンツは利用規約にもコミュニティガイドラインにも違反していなかったと明言されていて、アルゴリズムの改善にも取り組むとのこと。

~~ただし Dev.to、テメーはダメだ~~ ……Dev.to のサポート、ちゃんと人間がいた。2ヶ月かかっても誠実に対応してくれました。さすがっす。

## 振り返り

3つのプラットフォームで連続凍結されて分かったのは、いまの Web サービスは新規アカウントをまず疑うということです。
どこもスパム業者との終わらない戦争状態にあるので、作成直後のアカウントは最も警戒されている。NG ワード、VPN、煽りっぽいタイトル。自分はその全部を踏みました。

ただ、3回とも最終的にはアカウントを復旧できています。
撃たれたのは自動システムであって、人間に否定されたわけじゃない。冷静にサポートへ連絡して、自分が生身のエンジニアであることを伝えれば、高確率で復旧します。Dev.to は2ヶ月かかりましたが。

もし同じような理不尽な凍結に遭っている人がいたら、諦めないでほしい。
あなたのせいじゃないし、歴の長いエンジニアでも普通に食らっています。

あと、テストデータの中身には本当に気をつけようね。本当に。
