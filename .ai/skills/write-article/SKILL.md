# 役割

ユーザーのたたき台メモを起点に、質問を重ねながら Zenn 記事を完成させる。
一度にすべてを完成させようとせず、**アウトライン → ドラフト → 質問 → 反映** のサイクルで進める。

## ワークフロー

### Step 1: 既存記事から構成パターンを把握する

`articles/` ディレクトリの既存記事を1〜2本読み、セクション構成のパターン（導入の書き出し方、本文の展開、まとめ方）を把握する。

※ 文体・トーンや Zenn 固有フォーマットのルールは AGENTS.md に定義済み。ここでは構成の参考にとどめる。

### Step 2: アウトラインを提示する

ドラフトを書く前に、以下を含むアウトライン（見出し構成案）をユーザーに提示して合意を得る。

- 読者ターゲット（誰に向けた記事か）
- 記事のゴール（読者に何を持ち帰ってほしいか）
- h2 / h3 レベルの見出し構成
- 各セクションで扱う内容の要約（1〜2行）

ユーザーの承認後に次のステップへ進む。

### Step 3: ファイルを作成してドラフトを書く

`make new-article` を実行して記事ファイルと画像ディレクトリを生成し、承認済みのアウトラインに沿ってドラフトを書く。

ドラフト作成の方針:

- メモに書かれた要素をすべて拾い、記事の骨格を作る
- 情報が不足している箇所には TODO コメント (`<!-- TODO: ... -->`) を残す
- Step 1 で把握した構成パターンを参考にする

### Step 4: 質問してドラフトを膨らませる

ドラフト作成後、以下のルールで質問する。

- 1ターンに1件ずつ質問する
- 最大5回まで質問する
- 前の回答を踏まえて次の質問を決める
- 記事のコアとなる箇所から優先的に質問する

質問の観点（必要なものを選ぶ）:

- 読者ターゲットやゴールがドラフトの内容と合っているか
- 抽象的な主張を裏付けるエピソード・数字はあるか
- リンクや画像で補強できる箇所はあるか
- 記事のトーン（技術レビュー寄りか、個人的な体験談寄りか）
- ボリューム感（追加・削除したいセクションはあるか）

### Step 5: 回答を反映して記事を更新する

- TODO コメントを実際のコンテンツに置き換える
- 新しい情報をもとにセクションを加筆・修正する
- 不要になった箇所を削除してテンポを整える

### Step 6: 仕上げ確認

記事がほぼ完成したら以下を確認する。

- TODO コメントが残っていないか
- 導入と結論が対応しているか（導入で提示した問いに結論で答えているか）
- 見出し構成が一貫しているか（粒度のばらつきがないか）
- セクション間の流れが自然か（唐突な話題転換がないか）

## 禁止事項

- ユーザーが依頼していない内容を勝手に追加しない
- 質問をスキップして一度にすべてを完成させようとしない
