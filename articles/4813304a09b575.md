## TL;DR

Claude Code 用のステータスラインツール **Claude Nano Line** を作りました。
シェルの `$PS1` のようにフォーマット文字列で表示内容を自由に制御でき、API 使用率・リセット時間・コンテキスト・Git ブランチなどをリアルタイムに表示します。

- ワンライナーでインストールしてすぐ使える
- 単一の Python ファイルで外部依存なし
- 組み込みテーマですぐ使える、フォーマット文字列で細かくカスタマイズもできる

https://github.com/HappyOnigiri/ClaudeNanoLine

**クイックスタート:**

```bash
# 1. インストール
curl -fsSL https://raw.githubusercontent.com/HappyOnigiri/ClaudeNanoLine/main/setup.sh | bash

# 2. 好みのフォーマットを設定（省略するとデフォルト表示）
export CLAUDE_NANO_LINE_FORMAT="{5h_pct} {model} {cwd} ({branch})"

# 3. Claude Code を起動するとステータスラインに反映
claude
```

## Claude Code のステータスラインとは

Claude Code にはステータスライン機能があり、画面下部に任意の情報を常時表示できます。
外部コマンドの出力をステータスラインに流し込む仕組みで、`settings.json` にコマンドを登録するだけで有効になります。

API の残量やコンテキストの消費量など、作業中に気になる情報をチラ見できるので、ヘビーユーザーにはかなり便利な機能です。

## なぜ作ったのか

Claude Code のステータスラインには既存の有名なツールがあり、自分も使っていました。
UI がリッチで導入もかんたん。……なんですが、使い込んでいくうちにだんだん不満が溜まってきました。

**スペースを 1 つ挟みたいだけなのに UI を何ステップも操作しないといけない。**

(ﾟДﾟ)ﾊｧ?

リセットまでの残り時間を `2d` のような短縮表示にしたい？ → **フォーマット指定できません。**

(´；ω；｀)そんな〜

さらに、利用量の API 取得に失敗したときにすぐリトライする設計になっていて、**レート制限がいつまで経っても解除されない**という問題にも遭遇。
「ちょっと待てば回復するのに、自分でリトライし続けて自分の首を絞めている」という地獄のループ。

加えて、既存ツールは `npx` や `bunx` 経由で動作するため、実行のたびに Node.js のランタイムが走り、そこそこ負荷が高いという声もあります。
ステータスラインは定期的に実行されるので、軽さは正義。

もうこれは自分で作るしかない。
普段コードを書いている人なら `$PS1` のようなフォーマット記法は馴染み深いはず。
**その感覚で書けるステータスラインツール**を目指して Claude Nano Line を作りました。

## 主な機能

### フォーマット文字列駆動

環境変数 `CLAUDE_NANO_LINE_FORMAT` にフォーマット文字列を設定するだけ。
表示内容・色・条件付き表示をこれ一本で制御できます。

```bash
# シンプルに API 使用率とモデル名だけ表示
export CLAUDE_NANO_LINE_FORMAT="{5h_pct} {7d_pct} {model}"
```

```bash
# 色としきい値を細かく設定
export CLAUDE_NANO_LINE_FORMAT="{5h_pct|color:green,warn-color:yellow,alert-color:red,warn-threshold:70,alert-threshold:90}"
```

```bash
# 70% 未満は非表示にして、気になるときだけ表示
export CLAUDE_NANO_LINE_FORMAT="{5h_pct|hide-under:70} {model}"
```

スペースを入れたければ文字列にスペースを書くだけ。当たり前だけど、これがやりたかったんです。

### 豊富なプレースホルダー

表示できる情報は多岐にわたります。

| カテゴリ     | プレースホルダー例           | 説明                                     |
| ------------ | ---------------------------- | ---------------------------------------- |
| API 使用率   | `5h_pct`, `7d_pct`           | 5 時間枠・7 日枠の使用率                 |
| リセット時間 | `5h_reset`, `7d_reset`       | 制限リセットまでの残り時間               |
| リセット日時 | `5h_reset_at`, `7d_reset_at` | リセット日時（タイムゾーン対応）         |
| コンテキスト | `ctx_pct`, `ctx_tokens`      | コンテキスト使用率・トークン数           |
| Git          | `branch`, `branch_dirty`     | ブランチ名（ダーティマーク付きも可）     |
| パス         | `cwd`, `cwd_short`           | カレントディレクトリ                     |
| モデル       | `model`                      | Haiku / Sonnet / Opus を自動判定・色分け |

リセット時間の `2d` 表示？ `{5h_reset|unit:d}` で一発です(｀・ω・´)

### 組み込みテーマ

「細かい設定は後でいいからとりあえず使いたい」という方、安心してください。
テーマを選ぶだけで OK です。

```bash
export CLAUDE_NANO_LINE_THEME=ocean   # 青/シアンパレット
```

`ocean`・`forest`・`sunset`・`minimal`・`nerd` の 5 種類を用意しています。

![テーマプリセットの一覧](/images/4813304a09b575/theme-presets.png)

### 軽量な実行モデル

Python スクリプトを 1 回実行するだけで完結します。
標準ライブラリだけで動く単一ファイルなので、起動オーバーヘッドは最小限です。

### 堅牢なキャッシュとエラーハンドリング

API レスポンスは 360 秒間キャッシュし、**取得に失敗した場合も 360 秒間はリトライしません**。
エラー発生時は原因を詳細に表示し、ログファイルにも記録されるので、トラブルシューティングもしやすい設計です。

## インストール

### ワンライナー（推奨）

```bash
curl -fsSL https://raw.githubusercontent.com/HappyOnigiri/ClaudeNanoLine/main/setup.sh | bash
```

これだけ。
実行すると変更内容の差分が表示され、確認プロンプトが出ます。
既存の `settings.json` を壊す心配はありません。

### 手動インストール

```bash
cp claude-nano-line.py ~/.claude/
chmod +x ~/.claude/claude-nano-line.py
```

`~/.claude/settings.json` に以下を追加します。

```json:~/.claude/settings.json
{
  "statusLine": {
    "type": "command",
    "command": "python3 ~/.claude/claude-nano-line.py"
  }
}
```

動作要件は **Python 3.7 以上**です。
macOS・Windows（Git Bash / WSL）・Linux に対応しています。

## カスタマイズ例

いくつか実用的な設定例を紹介します。
`$PS1` をいじったことがある人なら、見ればだいたいわかるはず。

### リセット時間を時間単位で表示

```bash
export CLAUDE_NANO_LINE_FORMAT="{5h_pct} ({5h_reset|unit:h,digits:2}) {model}"
```

### Git ブランチをダーティ状態で色分け

```bash
export CLAUDE_NANO_LINE_FORMAT="{5h_pct} {model} {branch_dirty|color:cyan,dirty-color:yellow}"
```

### エラー時に表示を隠す

```bash
export CLAUDE_NANO_LINE_FORMAT="{5h_pct|on-error:hide} {model}"
```

### シェルコマンドを埋め込む

プレースホルダーだけでは物足りない方は、任意のシェルコマンドの出力も埋め込めます。
好きなコマンドで好き放題カスタマイズしちゃってください。

```bash
export CLAUDE_NANO_LINE_FORMAT="{cmd:date +%H:%M|color:cyan} {model}"
```

詳しいオプションは [README（日本語）](https://github.com/HappyOnigiri/ClaudeNanoLine/blob/main/README.ja.md) を参照してください。
